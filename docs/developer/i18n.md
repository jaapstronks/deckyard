# Internationalization (i18n)

This guide covers the UI internationalization system for the presentation system.

## Directory Structure

```
client/i18n/
├── manifest.json           # List of available locales
├── shared.json             # Language-agnostic strings (208 keys)
├── en/                     # English (reference language)
│   ├── index.json          # Merged file (loaded by app)
│   ├── common.json         # Common UI strings
│   ├── auth.json           # Login, forgot/reset password
│   ├── editor.json         # Editor interface
│   ├── list.json           # Presentation list, libraries
│   ├── share.json          # Sharing, viewer, moderation
│   ├── settings.json       # Settings, admin panel
│   ├── presenter.json      # Presenter mode
│   └── slide-types.json    # Slide type labels/options
├── nl/                     # Dutch
│   └── (same structure)
├── de/, fr/, es/, pt/, da/, sv/, no/
│   └── (same structure)
```

## Module Organization

| Module | Categories | Purpose |
|--------|------------|---------|
| `shared.json` | (all languages) | Color names, numbers, symbols, brand names |
| `common.json` | common, app, follow | Generic UI actions |
| `auth.json` | login, forgotPassword, resetPassword | Authentication flows |
| `editor.json` | editor | Editor interface |
| `list.json` | list, slideLibrary, imageLibrary | Lists and libraries |
| `share.json` | share, shareViewer, moderate | Sharing features |
| `settings.json` | settings, admin | User/admin settings |
| `presenter.json` | presenter, notesJoin | Presenter mode |
| `slide-types.json` | slideType | Slide type labels |

## Key Conventions

Use dot-notation keys with category prefix:

```javascript
t('common.save', 'Save')
t('editor.share.title', 'Share presentation')
t('slideType.textSlide.label', 'Text Slide')
```

## Using Translations

Import and call `t()`:

```javascript
import { t } from '/client/lib/ui-i18n.js';

// Simple translation
const label = t('common.save', 'Save');

// With interpolation
const msg = t('list.count', '{count} presentations', { count: 5 });
```

The second argument is the English fallback, used when:
- Translation is missing for current locale
- File fails to load

## Adding New Strings

1. Add the key to the appropriate English module file (e.g., `en/editor.json`)
2. Run the merge script: `node scripts/i18n-merge.js`
3. Run the sync script to add placeholders to other languages: `node scripts/i18n-sync.js`

## Scripts

| Script | Purpose |
|--------|---------|
| `node scripts/i18n-merge.js` | Rebuild `index.json` files from modules |
| `node scripts/i18n-sync.js` | Fill missing keys with English placeholders |
| `node scripts/i18n-validate.js` | Check for errors, missing keys, file sizes |
| `node scripts/i18n-extract.js` | Extract keys from source code |
| `node scripts/i18n-modularize.js` | One-time script to split flat files |

## Workflow

### After editing module files:
```bash
node scripts/i18n-merge.js
```

### After adding new keys to English:
```bash
node scripts/i18n-sync.js
```

### To validate all files:
```bash
node scripts/i18n-validate.js
```

## File Size Limits

All module files must stay under **600 lines**. If a module grows too large, consider splitting it into sub-modules.

Current largest file: `slide-types.json` (~460 lines)

## Shared Strings

The `shared.json` file contains 208 keys that are identical across all languages:
- Color names: "lime", "mist", "gray"
- Numbers: "1", "2", "3", etc.
- Symbols: "—" (em dash)
- Brand names: "Notion", "Markdown"

These are loaded once and merged with language-specific translations.

## RTL (Right-to-Left) Text Support

Slide content supports RTL languages (Arabic, Hebrew, Persian, Urdu) automatically. Users can type RTL text in any slide field and it will render correctly.

### How it works

All text elements in slide HTML use `dir="auto"`, which lets the browser detect text direction from the first strong directional character. This means:

- **Arabic/Hebrew text** → renders right-to-left
- **English/Dutch text** → renders left-to-right
- **Mixed content** → each element determines its own direction

No manual toggle is needed. Users just type in their preferred language.

### Technical implementation

1. **Slide type templates** (`shared/slide-types/types/*.js`): All text elements (`<h1>`, `<h2>`, `<p>`, `<div>` with text) include `dir="auto"`

2. **Markdown renderer** (`shared/markdown.js`): Generated elements (`<p>`, `<ul>`, `<ol>`, `<li>`, `<th>`, `<td>`) include `dir="auto"`. Code blocks keep `dir="ltr"` to preserve syntax direction.

3. **CSS** (`client/styles/slides/`): Uses `text-align: start` instead of `text-align: left` for direction-aware alignment

4. **HTML export** (`server/export/html.js`): Sets `dir` attribute on `<html>` element based on document language via `getDocDir()` from `server/utils/doc-lang.js`

### Supported RTL language codes

| Code | Language |
|------|----------|
| ar | Arabic |
| he | Hebrew |
| fa | Persian (Farsi) |
| ur | Urdu |

### Adding a new slide type

When creating a new slide type, ensure all user-facing text elements include `dir="auto"`:

```javascript
renderHtml: (content) => `
  <div class="slide slide-example">
    <h2 class="heading" dir="auto">${esc(content?.title)}</h2>
    <p class="body" dir="auto">${esc(content?.body)}</p>
  </div>
`
```
